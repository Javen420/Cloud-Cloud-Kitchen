── Shared Utilities ─────────────────────────────────────
config.py   — Pydantic Settings (loads .env automatically)
database.py — Supabase connection helper
redis.py    — Redis connection helper

Import these in your microservice to use them.

── Config / Settings (config.py) ───────────────────────
config.py uses Pydantic BaseSettings to load environment
variables into a typed Python object. It reads from your
.env file automatically.

  from shared.config import Settings
  settings = Settings()

Now you can access any env variable as an attribute:

  settings.supabase_url       # from SUPABASE_URL in .env
  settings.redis_addr         # defaults to "localhost:6379"
  settings.rabbitmq_url       # defaults to "amqp://guest:guest@localhost:5672/"
  settings.port               # defaults to 8080

All available fields and their defaults:

  supabase_url          ""
  supabase_key          ""
  supabase_jwt_secret   ""
  redis_addr            "localhost:6379"
  redis_password        ""
  rabbitmq_url          "amqp://guest:guest@localhost:5672/"
  google_routes_api_key ""
  stripe_secret_key     ""
  outsystems_menu_url   ""
  order_fulfilment_url  "http://localhost:8081"
  assign_kitchen_url    "http://localhost:8083"
  order_fulfilment_coord_url "http://localhost:8084"
  assign_driver_url     "http://localhost:8086"
  eta_tracking_url      "http://localhost:8087"
  notification_url      "http://localhost:8090"
  port                  8080

To set these, add them to your .env file in UPPER_CASE:

  SUPABASE_URL=https://your-project.supabase.co
  SUPABASE_KEY=your-key
  RABBITMQ_URL=amqp://guest:guest@rabbitmq:5672/

Any env vars not listed above are silently ignored (extra = "ignore").

── Importing (only add what your service needs) ─────────

  from shared.database import get_supabase
  db = get_supabase()

  from shared.redis import get_redis
  r = get_redis()

── Supabase (permanent data: orders, drivers, users) ───

  # SELECT * FROM drivers WHERE status = 'available'
  db.table("drivers").select("*").eq("status", "available").execute()

  # INSERT
  db.table("orders").insert({"item": "burger", "status": "pending"}).execute()

  # UPDATE orders SET status = 'completed' WHERE id = 123
  db.table("orders").update({"status": "completed"}).eq("id", 123).execute()

  # DELETE FROM orders WHERE id = 123
  db.table("orders").delete().eq("id", 123).execute()

── Redis (temporary/fast data: caching, live locations) ─

  # Store a value
  r.set("driver:123:location", "1.35,103.82")

  # Get a value
  r.get("driver:123:location")

  # Set with expiry (e.g. cache ETA for 60 seconds)
  r.setex("eta:order:456", 60, "15 mins")

  # Delete
  r.delete("driver:123:location")

--AMQP --
copy and paste this into main.py of any service that is
directly sending amqp requests to notifications service

if your service is also connecting to the db or redis,
just add it within the lifespan function

```
from contextlib import asynccontextmanager
from fastapi import FastAPI
from shared.config import Settings
from shared.amqp import AMQPPublisher

settings = Settings()
publisher = AMQPPublisher()

@asynccontextmanager
async def lifespan(app: FastAPI):
    await publisher.connect(settings.rabbitmq_url)
    app.state.publisher = publisher
    yield
    await publisher.close()

app = FastAPI(lifespan=lifespan)
```
